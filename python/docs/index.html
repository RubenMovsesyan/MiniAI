PyTorch Track

Rebuilding the from-scratch MNIST classifier in PyTorch — then shipping it as ONNX.
Code: python/ · Math background: ~/Documents/ml_training

1. What this is

The C++/CUDA project in ../src built a neural network from nothing — matrix kernels, activations, a fused softmax⊕cross-entropy gradient, an SGD engine — purely to understand the math. That worked: a 784→128→10 network hits ~97.6% test accuracy on MNIST. The full derivation lives in the math docs.

That exercise is done. This track switches to PyTorch — the framework you'd actually use in practice. The point now is to learn PyTorch, so the model itself is left for you to write. Everything around it is already set up.

The goal. Reproduce the same 784→128→10 classifier and the same ~97.6% test accuracy — but in ~30 lines of PyTorch instead of a few thousand lines of CUDA. Then export it to ONNX and run it through an inference engine.

2. Setup & layout

System Python is 3.14 and externally-managed (PEP 668), so a virtual environment is mandatory. setup.sh handles it:

cd python
./setup.sh                     # venv + torch (cu128) + deps
source .venv/bin/activate
python data.py                 # self-check the IDX loader
python main.py                 # runs, stops at the first TODO

Run it before writing any torch. python main.py loads the data, prints the device and batch shapes, then raises NotImplementedError at build_model. That confirms the plumbing works — you only ever debug your own code.

3. The target

Here's the curve to beat, measured on the C++ engine (lr 0.1, batch 100). If your PyTorch model doesn't land here, something is wrong — the two implementations should agree closely.

Target: test accuracy → 97.6%, train cross-entropy → ~0.05 over 15 epochs. Detailed in the PyTorch guide.
